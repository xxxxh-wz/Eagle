## 2. LocateAnything 基础部署

```bash
cd /home/user/code
git clone https://github.com/NVlabs/Eagle.git eagle
cd /home/user/code/eagle/Embodied

conda create -y -n locateanything python=3.10 pip

conda run -n locateanything python -m pip install \
  torch==2.11.0+cu130 torchvision==0.26.0+cu130 \
  --index-url https://download.pytorch.org/whl/cu130

conda run -n locateanything python -m pip install -e .
```

说明：

- 官方 README 的基础安装入口是 `pip install -e .`。
- `pyproject.toml` 没有直接声明 `torch`，所以先手动安装 CUDA 版 PyTorch。
- 本机只有 CUDA 13.0 nvcc，因此最终使用 `cu130` PyTorch wheel，避免后续编译 MagiAttention 时 CUDA mismatch。

## 3. MagiAttention 加速依赖部署

Eagle 官方 Embodied README 建议 Hopper / Blackwell GPU 使用 MagiAttention。H200 属于 Hopper，适合安装。

```bash
cd /home/user/code
git clone https://github.com/SandAI-org/MagiAttention.git
cd /home/user/code/MagiAttention

git checkout v1.0.5
git submodule update --init --recursive

conda run -n locateanything python -m pip install -r requirements.txt
conda run -n locateanything python -m pip install debugpy
```

构建安装：

```bash
cd /home/user/code/MagiAttention

env \
  MAGI_ATTENTION_WORKSPACE_BASE=/home/user/code \
  TORCH_CUDA_ARCH_LIST=9.0 \
  NVCC_THREADS=4 \
  MAGI_ATTENTION_PREBUILD_FFA_JOBS=16 \
  conda run -n locateanything python -m pip install --no-user --no-build-isolation .
```

构建完成后，固化 conda 环境变量：

```bash
conda env config vars set -n locateanything \
  MAGI_ATTENTION_WORKSPACE_BASE=/home/user/code \
  TORCH_CUDA_ARCH_LIST=9.0
```

修复基础依赖约束：

```bash
conda run -n locateanything python -m pip install \
  'numpy>=1.25,<2' \
  'fsspec>=2023.1.0,<=2026.2.0' \
  'setuptools>=78.1.1,<82'
```

## 4. 踩坑记录

### 4.1 PyTorch CUDA 版本必须和 nvcc 匹配

一开始安装了：

```bash
torch==2.11.0+cu128
```

但系统 `nvcc` 是 CUDA 13.0。构建 MagiAttention 最后阶段报错：

```text
RuntimeError: The detected CUDA version (13.0) mismatches the version that was used to compile PyTorch (12.8)
```

解决：

```bash
conda run -n locateanything python -m pip install --force-reinstall \
  torch==2.11.0+cu130 torchvision==0.26.0+cu130 \
  --index-url https://download.pytorch.org/whl/cu130
```

### 4.2 MagiAttention 官方 requirements 缺少 debugpy

构建时报错：

```text
ModuleNotFoundError: No module named 'debugpy'
```

解决：

```bash
conda run -n locateanything python -m pip install debugpy
```

### 4.3 默认 JIT cache 写到只读 HOME

构建时报错：

```text
OSError: [Errno 30] Read-only file system: '/home/user/.cache/magi_attention'
```

原因：MagiAttention 默认使用 `$HOME/.cache/magi_attention`。当前环境里该路径不可写。

解决：设置可写的工作目录。

```bash
MAGI_ATTENTION_WORKSPACE_BASE=/home/user/code
```

并写入 conda 环境变量：

```bash
conda env config vars set -n locateanything MAGI_ATTENTION_WORKSPACE_BASE=/home/user/code
```

### 4.4 构建时无法自动探测 CUDA arch

构建时报错：

```text
IndexError: list index out of range
```

原因：构建阶段 NVML 初始化失败，PyTorch 无法自动生成 CUDA arch flags。

解决：H200 是 Hopper，显式指定：

```bash
TORCH_CUDA_ARCH_LIST=9.0
```

### 4.5 setuptools 版本两边都有约束

MagiAttention 官方 requirements 要：

```text
setuptools>=78.1.1
```

PyTorch 2.11.0 wheel 元数据要求：

```text
setuptools<82
```

最终兼容选择：

```bash
setuptools==81.0.0
```

### 4.6 pip install 可能尝试写入只读 user site

构建成功后安装时报错：

```text
OSError: [Errno 30] Read-only file system: '/home/user/.local/lib'
Defaulting to user installation because normal site-packages is not writeable
```

解决：使用提升权限或确保可写 conda env，并加 `--no-user`：

```bash
conda run -n locateanything python -m pip install --no-user --no-build-isolation .
```

### 4.7 pip check 的 decord 提示

当前 `pip check` 有一个提示：

```text
decord 0.6.0 is not supported on this platform
```

本次 LocateAnything 单图推理不走视频解码路径，实测不影响 `nvidia/LocateAnything-3B` 图像定位推理。

## 7. 迁移到另一台服务器的建议顺序

1. 先确认 GPU 是 Hopper / Blackwell，且 `nvcc --version` 与 PyTorch CUDA wheel 匹配。
2. 如果系统只有 CUDA 13.0，优先使用 `torch==2.11.0+cu130`。
3. 先安装 LocateAnything 并跑通无加速推理。
4. 再安装 MagiAttention `v1.0.5`。
5. 构建 MagiAttention 时显式设置：

```bash
MAGI_ATTENTION_WORKSPACE_BASE=/home/user/code
TORCH_CUDA_ARCH_LIST=9.0
NVCC_THREADS=4
MAGI_ATTENTION_PREBUILD_FFA_JOBS=16
```

6. 安装完成后复跑 `smoke_test_locateanything.py`。
7. 验收标准：
   - `import magi_attention` 成功。
   - `torch.version.cuda` 与系统 `nvcc` 主版本匹配。
   - 推理日志不再出现 `magi_attention not available`。
   - 风筝图查询能返回可解析 `<box>...</box>`。
